Overview

Project Goals: Reduce domestic energy consumption using smart meter data and engage users positively with climate action.

Background: This project leverages smart meter data and AI insights to reduce confusion and promote actionable energy-saving habits. Supported by Gengame Ltd., the project tackles key challenges in user engagement and behavior change.

Smart meters provide real-time energy usage data, enabling households to:

  • Adopt energy-efficient behaviors.
  • Invest in home efficiency improvements.
  • Shift energy usage to reduce peak demand.
  • Engage with renewable energy options.

The UK governmentâ€™s goal is to have all households offered smart meters by 2025, making this a critical time to drive meaningful climate-positive action.

Coursework Stages

The project is divided into three distinct stages to structure the design process effectively.

  • Discovery: Define the problem space, explore user goals, and create a UX Vision Statement.
  • Design: Ideate app features, storyboard user flows, and build low-fidelity prototypes.
  • Delivery: Develop mid-fidelity Figma prototypes showcasing user-centered solutions.

Stage 1: Discovery

  • Analyze how smart meter data can reduce energy consumption.
  • Identify key obstacles and user pain points.
  • Deliverables: UX Vision Statement, Design Principles.

Stage 2: Design

  • Brainstorm app features and map user needs to functionality.
  • Create low-fidelity prototypes for key user flows.

Stage 3: Delivery

  • Present a mid-fidelity prototype developed in Figma.
  • Ensure usability and compliance with accessibility standards.

Submission Requirements

Prepare a detailed A3 landscape report that clearly communicates your design process and outcomes.

  • Title page with clickable Figma prototype link.
  • Discovery insights and key user needs.
  • Wireframes and sketches showcasing user flows.
  • Evaluation and reflection on prototyping outcomes.

Assessment Criteria

Your work will be evaluated based on the following criteria:

  • Knowledge: Demonstrates understanding of UX design principles.
  • Application: Effectively utilizes research and prototyping techniques.
  • Design Outcome: Offers a novel and user-friendly solution.
  • Communication: Clear and professional presentation of ideas.

Evaluation Process

Prototype evaluation is a critical step to refine your design. Use these methods:

  • User Testing: Gather feedback from target users using low-fidelity prototypes.
  • Scenario Walkthroughs: Simulate user interactions to identify usability issues.
  • Iterative Refinement: Incorporate feedback to improve design effectiveness.

Key Steps:

  • Plan evaluation sessions using the Question Plan Test Reflect framework.
  • Document user feedback with photos or notes.
  • Analyze findings to identify patterns and actionable insights.
  • Sketch alternative designs for unresolved issues and test them iteratively.

Ensure all participants provide informed consent before evaluation begins.
